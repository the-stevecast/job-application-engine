# Resume Bullet Bank

> **How the agent uses this file:** Selects and tailors bullets for each application based on the role strategy profile. The agent never invents experience — it only draws from bullets listed here. The more bullets you have, the better the agent can match to any role.
>
> **Bullet format rule:** Every bullet must follow **Ownership verb + System/Process + Impact**.
> - Start with a strong ownership verb: Built, Designed, Led, Established, Launched, Defined, Created, Architected, Grew, Reduced, Restructured
> - State what was built, designed, or implemented
> - End with the operational result (faster, reduced, eliminated, improved, enabled, saved, grew)
> - 1-2 lines max. If it runs to 3 lines, split it or cut words.
> - No soft verb openers: Helped, Supported, Assisted, Participated, Worked on
>
> **Timeline accuracy:** Note the date range for each role. Do not imply capabilities started earlier or later than they did.
>
> **How to populate:** Run `/setup` or `/setup bullets`. Or fill in directly. The agent will also suggest additions in `revision-notes.md` after each application.
